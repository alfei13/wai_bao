# 采集端接口文档

## 设计说明

- 所有接口均由**采集端主动调用业务端**
- 协议：HTTP
- 数据格式：JSON
- 时间格式：`yyyy-MM-dd HH:mm:ss`

---

## 接口列表

| 序号 | 接口 | 方法 | 方向 | 说明 |
|---|---|---|---|---|
| 1 | `/api/collect/rule` | GET | 采集端 → 业务端 | 拉取采集规则 |
| 2 | `/api/collect/data` | POST | 采集端 → 业务端 | 上传采集结果 |
| 3 | `/api/collect/status` | POST | 采集端 → 业务端 | 上报任务状态 |

---

## 1. 拉取采集规则

采集端启动或定时任务触发时调用。

**GET** `/api/collect/rule`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `device_id` | string | 是 | 采集端设备标识 |
| `platform` | string | 否 | 指定平台，为空返回全部 |

### 请求示例

```
GET /api/collect/rule?device_id=DEV001&platform=caiyiluo
```

### 响应结果

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "task_id": "T20260703200001",
    "platforms": ["caiyiluo", "darunfa"],
    "keywords": ["鸡蛋", "猪肉", "白菜", "土豆", "番茄"],
    "max_items": 2,
    "cron": "0 8 * * *"
  }
}
```

### data 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `task_id` | string | 任务ID，采集端上报时需带回 |
| `platforms` | array | 要采集的平台列表 |
| `keywords` | array | 关键词列表 |
| `max_items` | int | 每个关键词最多采集条数 |
| `cron` | string | 定时规则，为空表示立即执行一次 |

---

## 2. 上传采集结果

采集端完成单个平台采集后调用。

**POST** `/api/collect/data`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `task_id` | string | 是 | 任务ID |
| `platform` | string | 是 | 平台：`caiyiluo`/`darunfa`/`shixing` |
| `collected_at` | string | 是 | 采集完成时间 |
| `items` | array | 是 | 价格数据列表 |

### items 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | 是 | 搜索关键词 |
| `name` | string | 是 | 商品名称 |
| `price` | number | 是 | 价格，单位元 |

### 请求示例

```json
{
  "task_id": "T20260703200001",
  "platform": "caiyiluo",
  "collected_at": "2026-07-03 20:35:30",
  "items": [
    { "keyword": "鸡蛋", "name": "鲜鸡蛋", "price": 18.58 },
    { "keyword": "鸡蛋", "name": "白壳土鸡蛋", "price": 20.38 },
    { "keyword": "白菜", "name": "大白菜", "price": 0.78 }
  ]
}
```

### 响应结果

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "saved": 36
  }
}
```

## 3. 上报任务状态

采集端完成整个任务或某个平台失败后调用。

**POST** `/api/collect/status`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `task_id` | string | 是 | 任务ID |
| `platform` | string | 是 | 平台 |
| `status` | string | 是 | 状态：`success`/`partial`/`failed` |
| `total` | int | 是 | 应采集条数 |
| `done` | int | 是 | 成功条数 |
| `error` | string | 否 | 失败原因 |

### 请求示例

```json
{
  "task_id": "T20260703200001",
  "platform": "caiyiluo",
  "status": "success",
  "total": 36,
  "done": 36,
  "error": ""
}
```

### 响应结果

```json
{
  "code": 0,
  "msg": "ok"
}
```

---

## 公共响应码

| code | 说明 |
|---|---|
| 0 | 成功 |
| 4001 | 参数错误 |
| 4002 | 任务不存在 |
| 5001 | 服务端内部错误 |

## 数据库表设计

```sql
CREATE TABLE price_records (
  id           BIGINT PRIMARY KEY AUTO_INCREMENT,
  task_id      VARCHAR(32)  NOT NULL COMMENT '任务ID',
  platform     VARCHAR(32)  NOT NULL COMMENT '平台',
  keyword      VARCHAR(64)  NOT NULL COMMENT '关键词',
  product_name VARCHAR(256) NOT NULL COMMENT '商品名',
  price        DECIMAL(10,2) NOT NULL COMMENT '价格(元)',
  collected_at DATETIME     NOT NULL COMMENT '采集时间',
  created_at   TIMESTAMP    DEFAULT CURRENT_TIMESTAMP COMMENT '入库时间',
  INDEX idx_task (task_id),
  INDEX idx_platform_time (platform, collected_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='价格采集记录';
```

---

## Android 端调用流程

1. 启动时：`GET /api/collect/rule` 拉取规则
2. 按 `platforms` 和 `keywords` 执行采集
3. 单个平台完成后：`POST /api/collect/data` 上传数据
4. 全部完成后：`POST /api/collect/status` 上报状态
